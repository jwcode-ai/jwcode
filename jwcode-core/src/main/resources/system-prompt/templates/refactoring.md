# Task Template: Refactoring

## Workflow
```
Explore → Architect → Coder → Tester → Reviewer → Report
```

## Steps
1. **Explore**: Analyze current code, identify issues
2. **Architect**: Design refactoring plan
3. **Coder**: Execute refactoring in small, safe steps
4. **Tester**: Run full test suite to verify no regression
5. **Reviewer**: Review refactoring quality
6. **Orchestrator**: Generate report

## Acceptance Criteria
- [ ] Code is cleaner and more maintainable
- [ ] All existing tests pass
- [ ] No behavior change (unless intentional)
- [ ] Performance is not degraded
- [ ] Refactoring plan is documented
